Invoice Fields Reference

This reference shows all the fields you can include when sending invoice data to our API. Each field has a clear description and example to help you understand what information to provide.

Document Information

Field Name Type Required Description Example
Document Id string ✅ The unique identifier for your invoice. This should be unique for each invoice you send. INV-2024-001234
Document Type string ✅ The type of document you're sending. Use the appropriate code for your document type. tax_invoice
Issue Date string ✅ The date when the invoice was issued. Use YYYY-MM-DD format. 2024-01-15
Due Date string ⚪ The date when payment is due. Use YYYY-MM-DD format. 2024-02-15
Currency string ✅ The currency code for all amounts in the invoice. Use 3-letter ISO currency codes. USD, EUR, SAR

Seller Information

Field Name Type Required Description Example
Seller Name string ✅ The legal name of your business or organization issuing the invoice. Acme Corporation Ltd
Seller VAT Number string ✅ Your business VAT registration number. Format varies by country. GB123456789 (UK), 311111111101114 (Saudi)
Seller Address object ✅ Complete address information for your business. {"street": "123 Business St", "city": "London", "postal_code": "SW1A 1AA", "country": "GB"}
Seller Email string ⚪ Your business contact email address. billing@acme.com

Buyer Information

Field Name Type Required Description Example
Buyer Name string ✅ The name of the customer you're invoicing. Customer Company Inc
Buyer VAT Number string ⚪ The customer's VAT registration number, if they have one. GB987654321
Buyer Address object ✅ The customer's billing address information. {"street": "456 Customer Ave", "city": "Manchester", "postal_code": "M1 1AA", "country": "GB"}

Line Items

Field Name Type Required Description Example
Item Description string ✅ Clear description of the product or service being sold. Professional Consulting Services
Quantity number ✅ How many units of this item are being sold. 5
Unit Price number ✅ The price per unit before tax. 299.99
Tax Rate number ✅ The tax percentage applied to this item. 20 (for 20% VAT)

Totals

Field Name Type Required Description Example
Subtotal number ✅ Total amount before tax is added. 1000.00
Tax Amount number ✅ Total tax amount for the entire invoice. 200.00
Total Amount number ✅ Final amount the customer needs to pay including tax. 1200.00
Country-Specific Requirements
Saudi Arabia (ZATCA)
  • VAT Number must be 15 digits
  • QR Code automatically generated
  • Arabic translations supported
Malaysia (MyInvois)
  • GST Number required for businesses
  • Ringgit (MYR) currency for domestic
  • Automatic LHDN portal submission
United Kingdom
  • VAT Number format: GB + 9 digits
  • 20% standard VAT rate
  • HMRC Making Tax Digital compliance
Java SDK Implementation Guide

Step-by-step guide to integrate Complyance invoicing into your Java application

1Add SDK to Your Project

Add the SDK dependency to your project:

Maven
<dependency>
    <groupId>io.complyance</groupId>
    <artifactId>complyance-java-sdk</artifactId>
    <version>1.0.0</version>
</dependency>
Gradle
implementation 'io.complyance:complyance-java-sdk:1.0.0'
2Configure the SDK

Initialize the SDK with your API key:

import io.complyance.sdk.*;

// Configure your source information
Source source = new Source(
    SourceType.FIRST_PARTY,
    "your-system-id",
    "Your System Name",
    "1.0"
);

// Initialize with your API key
SDKConfig config = new SDKConfig(
    "your-api-key-here",
    Environment.SANDBOX,  // Use PRODUCTION for live
    Arrays.asList(source)
);

GETSUnifySDK.configure(config);
3Prepare Your Invoice Data

Create a Map with your invoice information:

Map<String, Object> invoice = new HashMap<>();

// Basic invoice information
Map<String, Object> header = new HashMap<>();
header.put("invoiceNumber", "INV-2024-001");
header.put("invoiceDate", "2024-01-15T10:30:00Z");
header.put("documentType", "tax_invoice");
header.put("currency", "USD");
invoice.put("header", header);

// Seller information (your business)
Map<String, Object> seller = new HashMap<>();
seller.put("name", "Your Business Name");
seller.put("vatNumber", "Your-VAT-Number");
seller.put("address", "Your Business Address");
invoice.put("seller", seller);

// Customer information
Map<String, Object> buyer = new HashMap<>();
buyer.put("name", "Customer Name");
buyer.put("address", "Customer Address");
invoice.put("buyer", buyer);

// Invoice line items
List<Map<String, Object>> items = new ArrayList<>();
Map<String, Object> item = new HashMap<>();
item.put("description", "Professional Services");
item.put("quantity", 1);
item.put("unitPrice", 1000.00);
item.put("taxRate", 20);
items.add(item);
invoice.put("items", items);
4Send Your Invoice

Submit your invoice to the tax authorities:

try {
    UnifyResponse response = GETSUnifySDK.pushToUnify(
        "your-system-id",           // Your source ID
        DocumentType.TAX_INVOICE,   // Document type
        "GB",                       // Country code (GB, SA, MY, etc.)
        Operation.SINGLE,           // Single document
        Mode.DOCUMENTS,             // Processing mode
        Purpose.PRODUCTION,         // Send to tax authority
        invoice                     // Your invoice data
    );
    
    if (response.getStatus().equals("success")) {
        String documentId = response.getData().get("documentId").toString();
        System.out.println("Invoice submitted successfully! ID: " + documentId);
    } else {
        System.out.println("Error: " + response.getMessage());
    }
    
} catch (SDKException e) {
    System.out.println("SDK Error: " + e.getMessage());
}
5Test Before Going Live

Always test your integration first:

// Test your invoice before sending to tax authority
UnifyResponse testResponse = GETSUnifySDK.pushToUnify(
    "your-system-id",
    DocumentType.TAX_INVOICE,
    "GB",
    Operation.SINGLE,
    Mode.DOCUMENTS,
    Purpose.VALIDATION,      // Test mode - won't send to tax authority
    invoice
);

if (testResponse.getStatus().equals("success")) {
    System.out.println("Invoice is valid and ready to send!");
} else {
    System.out.println("Invoice has issues: " + testResponse.getMessage());
}
